# Retrospective — v0.1.1

**Date:** 2026-03-25
**Milestone:** v0.1.1 — bash 3.2 compat, gist-01 install, subtree path fix
**PR:** #4 — merged to main (`c5bda1e`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- **Symlink tech debt fully resolved** — `_RIGOR_LIB_DIR` now points at the real subtree path; no more manual symlink repairs after each subtree pull
- **bash 3.2 compat fix clean** — `mapfile` → `while IFS= read -r` was a one-line change with no regressions
- **Gist published and linked** — one-command install is live and referenced in both README and how-to doc
- **Copilot review caught real issues** — both findings (pre-release link 404, premature CHANGE.md version) were valid; fixed same session
- **Issue doc discipline** — `docs/issues/` entry created for every Copilot review, per convention

## What Went Wrong

- **Multiple merge conflicts on feat/v0.1.1** — squash-merged PRs #2 and #3 left stale commits in the feature branch; each main sync triggered conflicts in CHANGE.md, memory-bank, gist README. Time lost to repeated conflict resolution.
- **Premature CHANGE.md versioning** — `[v0.1.1] — 2026-03-24` was written before the tag existed; Copilot caught it. Process gap: CHANGE.md must stay `[Unreleased]` until post-merge.
- **README tag link written before tag created** — same pattern as above; link would have 404'd while PR was open.
- **enforce_admins restore syntax issue** — `-f enabled=true` returns HTTP 422; correct call is `-X POST` with no body. Recurs across sessions.

## Process Rules Added

| Rule | Where |
|---|---|
| CHANGE.md must stay `[Unreleased]` while PR is open; finalize in `/post-merge` | PR template, Copilot findings doc |
| README releases table: no tag links until tag exists post-merge | PR template, Copilot findings doc |
| `enforce_admins` restore: use `-X POST` with no body (not `-f enabled=true`) | post-merge skill known failures |

## Decisions Made

- **No YAML checking in rigor-cli** — rigor-cli is shell-only. YAML hardcoded-IP prevention goes in lib-foundation `_agent_audit` (v0.3.11 spec written and handed to Codex).
- **gist-01 installed to `.rigor/`** — the subtree is placed at `.rigor/` in consumer repos (not `scripts/lib/foundation/`). `bin/rigor` wrapper delegates to `.rigor/bin/rigor`.
- **core.hooksPath compatibility** — `_install_pre_commit` resolves hooks dir via `git rev-parse --git-path hooks` instead of hardcoding `.git/hooks`.

## Theme

v0.1.1 was a milestone close-out sprint — no new features, just getting rigor-cli to a stable, installable state. The main blocker was accumulated branch divergence from three prior squash-merged PRs, each requiring a manual conflict resolution before work could continue. Despite the overhead, every planned item shipped: bash 3.2 compatibility, the gist-01 one-command installer (now live at https://gist.github.com/wilddog64/81c767a0560e39c8d6e0f8bd9706973b), the symlink tech debt removal, and a how-to install guide. Copilot review added two small but correct doc fixes. The repo is now in a clean state for v0.1.2.
