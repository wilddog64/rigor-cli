# Retrospective — v0.1.3

**Date:** 2026-05-03
**Milestone:** `rigor review` subcommand — AI review via `_ai_agent_review`
**PR:** #6 — merged to main (`675f6e9`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- `_ai_agent_review` dispatch pattern in lib-foundation made the rigor-cli integration trivial — `_rigor_review` is ~6 lines
- Copilot found all 4 doc issues in the subtree; fixing upstream in lib-foundation and subtree-pulling is the clean path
- BATS for `_agent_lint` `.js`/`.md` globs were straightforward to add once the `etc/agent/` path and mock function pattern were understood
- Branch tracking bug (fix/copilot-deny-tool-patterns pointing to origin/main) caught before any accidental push damage

## What Went Wrong

- Codex updated the k3d-manager memory-bank instead of rigor-cli memory-bank during handoff — spec didn't name which repo's memory-bank to update
- `_mock_ai` variable-capture approach in BATS initially used a local variable (`captured="$*"`) which doesn't survive `run` subshell; fixed to file-based capture
- CHANGE.md in rigor-cli shipped with `[Unreleased]` (no version number) — required an extra admin push post-merge to version it
- README and copilot-instructions.md still referenced "three subcommands" and a branch URL instead of a tag after merge

## Process Rules Added

| Rule | Source |
|---|---|
| Handoff blocks for rigor-cli tasks must name rigor-cli memory-bank explicitly, not k3d-manager | Codex confusion incident |
| CHANGE.md must be pre-versioned (`[vX.Y.Z] — date`) before creating the PR, not left as `[Unreleased]` | Extra admin push post-merge |

## Decisions Made

- `_rigor_review` warns on missing `.github/copilot-instructions.md` but does not block — Copilot falls back to defaults, which is still useful
- lib-foundation subtree is the only path for doc/function fixes in `scripts/lib/foundation/` — never edit in-consumer
- `_ai_agent_review` gate is `AGENT_LINT_GATE_VAR` / `ENABLE_AGENT_LINT`, not `K3DM_ENABLE_AI` — the consumer-specific env var was removed from lib-foundation in v0.3.17

## Theme

v0.1.3 wired rigor-cli into the AI review pipeline that lib-foundation shipped in v0.3.17–v0.3.20. The feature itself was small; the work was mostly resolving Copilot's accurate observation that the subtree docs still referenced a removed feature gate (`K3DM_ENABLE_AI`) and a wrong test count. The pattern held: Copilot catches doc drift, fix goes upstream, subtree pull brings it in.
