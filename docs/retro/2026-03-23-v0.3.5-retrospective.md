# Retrospective — v0.3.5

**Date:** 2026-03-23
**Milestone:** doc-hygiene pre-commit hook
**PR:** #10 — merged to main (`2f895a99`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Copilot caught real bugs across 3 review rounds — global variable leak (`_DHC_STAGED`), false-negative file existence in staged mode, wrong test path (working-tree instead of index)
- Codex delivered all `.sh` fixes cleanly once specs had exact old/new code blocks
- `_dh_grep` abstraction cleanly separated staged vs working-tree read paths
- `git cat-file -e` guard correctly handles files deleted from working tree after staging
- Spec discipline held: every Codex task had a spec with exact commit message; no commits to `main`

## What Went Wrong

- **Three Copilot review rounds** — round 1 caught 4 findings, round 2 caught 3 more, round 3 caught 2 more. Root cause: each round addressed the previous findings but introduced new behavior that Copilot hadn't seen yet. Expected pattern for iterative `.sh` work.
- **Spec missing repo name** — initial Codex handoff for `v0.3.5-doc-hygiene-staged-content-read.md` omitted the repo name; Codex couldn't find the branch. Fixed by adding `Repo: lib-foundation` + fetch/checkout instructions.
- **Staged-content test was wrong** — the spec (and initial Codex implementation) wrote the staged-mode test to verify the working-tree path, not the index path. Copilot caught it. Test rewritten in round 2.
- **Code-fence false positive** — `doc_hygiene.sh` Check 2 greps entire `.md` files including fenced code blocks; plan docs containing `http://` in code examples will trip the hook on future edits. Deferred to v0.3.6.

## Process Rules Added

| Rule | Where |
|---|---|
| Specs targeting non-k3d-manager repos must name the repo explicitly | `/handoff` skill, `## Before You Start` template |
| Staged-mode tests must call function with no args from inside the repo, not with explicit path | BATS test pattern |

## Decisions Made

- `_DHC_STAGED` is `local` to `_doc_hygiene_check` — Bash dynamic scoping means `_dh_grep` still sees it; no global leakage
- `git cat-file -e :"$file"` replaces `[[ -f "$file" ]]` in staged mode — handles files deleted from working tree after staging
- Code-fence exclusion for Check 2 deferred to v0.3.6 — complexity not justified for this milestone; practical impact is low (spec files committed once, rarely re-staged)

## Theme

v0.3.5 was a refinement milestone. The doc-hygiene hook was introduced in PR #10 and immediately subjected to three rounds of Copilot review that progressively tightened the implementation: from portability fixes, to staged-vs-working-tree correctness, to local variable scoping. Each round was a legitimate finding — Copilot was doing its job. The resulting `doc_hygiene.sh` is now meaningfully more correct than the first draft. The lesson: pre-commit hooks that interact with the git index are subtle; BATS tests for staged behavior require care to actually exercise the staged path.
