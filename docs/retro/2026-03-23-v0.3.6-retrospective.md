## Decisions Made

- `--strip-fences` applied to Check 2 only (bare `http://`) — Checks 1, 3, 4 intentionally skip it (placeholder URLs and private IPs are equally wrong inside code blocks)
- Check 4 is warn-only (non-blocking) — same pattern as Check 3 (private IPs); cross-cluster DNS issues should surface at pre-commit but not block work
- `_dh_strip_fences` tracks `fence_char` to correctly close mismatched fence types — backtick fence closed by backtick only, tilde by tilde only

## Theme

v0.3.6 was a quality pass on the doc-hygiene hook. The code-fence false positive was a known limitation from v0.3.5 (a plan doc with `http://` in a code example triggered Check 2). The fix required a new awk helper and an optional flag on `_dh_grep`. Copilot then found the indented fence gap the spec missed — the awk pattern only matched column-1 fences, so list-item code blocks were still scanned. Three Codex commits, four Copilot findings, one process rule. The hook is now production-grade for real markdown docs.
