# Retrospective — v0.1.0

**Date:** 2026-03-24
**Milestone:** Initial release — rigor-cli standalone CLI
**PR:** #1 — merged to main (`f720184`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Codex implemented dispatcher + subtree + BATS in one clean commit — no rework needed on core functionality
- BATS 3/3 passed locally and in CI on first try
- Copilot found a real supply-chain issue (unpinned bats-core) — process rule added
- Branch protection, `enforce_admins`, CI required status checks all wired in the same session
- `copilot-instructions.md`, `.clinerules`, and `memory-bank/` all set up before PR merged — repo is agent-ready from day one

## What Went Wrong

- `main` branch was never initialized before branching — all commits landed on `feat/init`, causing `gh pr create` to fail with "No commits between main and feat/init". Had to push the initial commit as main manually.
- README went through three rewrites (bare → lib-foundation format → k3d-manager format) — should have confirmed format preference before writing
- `scripts/lib/foundation/.clinerules` in the subtree has stale `_detect_platform` return values — upstream lib-foundation debt, not caught before PR

## Process Rules Added

| Rule | Where to add |
|------|-------------|
| New repos: push an initial commit to `main` before branching | `/create-pr` pre-flight or repo bootstrap checklist |
| Confirm README format preference before writing | Ask once at the start of a new repo session |
| Pin `git clone` steps in CI to a version tag (`--branch vX.Y.Z`) | Spec template CI section |
| Scan subtree docs for contract mismatches on first PR | New repo checklist |

## Decisions Made

- Separate repo (`wilddog64/rigor-cli`) confirmed — not embedded in k3d-manager; widest audience for any Bash project
- bash-only implementation (no Python pre-commit framework dependency)
- Subtree prefix `.rigor/` for consumer installs — hidden directory signals vendor/read-only
- `mapfile` in `_rigor_shellcheck` accepted for v0.1.0 (CI on Ubuntu); tracked as bash 3.2 compat debt for v0.1.1
- Symlinks at `scripts/lib/foundation/system.sh` + `agent_rigor.sh` are functional workaround for subtree path depth — tracked as tech debt

## Theme

rigor-cli ships. The core idea — expose lib-foundation's three audit functions as a standalone CLI any Bash project can install — went from discussion to tagged v0.1.0 in a single session. Codex did the implementation cleanly; Copilot caught the supply-chain gap; Claude handled wiring. The repo is now fully agent-ready: branch protection, memory-bank, `.clinerules`, `copilot-instructions.md` all in place. Next: gist install script to make it one-command installable.
