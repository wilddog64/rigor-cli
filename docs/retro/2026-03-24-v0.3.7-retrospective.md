# Retrospective — v0.3.7

**Date:** 2026-03-24
**Milestone:** system.sh if-count reduction — `_run_command` + `_ensure_node`
**PR:** #13 — merged to main (`071c270e`)
**Participants:** Claude, Copilot

## What Went Well

- Pure extraction — no behavioral changes; `_run_command` and `_ensure_node` semantics are identical
- BATS suite remained green (11/11) after both extractions
- shellcheck passed with zero new warnings on first attempt
- Copilot findings were minor (indentation + missing header comment) — caught nothing functional
- Both helpers (`_run_command_handle_failure`, `_node_install_via_redhat`) are self-contained with ≤4 ifs each — no follow-on allowlist entries needed

## What Went Wrong

- Indentation regression: the extracted call in `_run_command` used 3-space indent instead of 4-space (2-space style); Copilot caught it — should have been caught by eye before push
- `_node_install_via_redhat` was added without a header comment, inconsistent with `_run_command_handle_failure` which had one — both helpers should have been treated uniformly

## Process Rules Added

| Rule | Where |
|---|---|
| New private helpers must have a `# <description>` header comment, matching existing helpers in the same file | lib-foundation code review checklist |

## Decisions Made

- `_run_command_handle_failure` takes `runner[@]` + `"$@"` (the full executed command) so it can reproduce the original failure log message faithfully — not just the prog name
- `_node_install_via_redhat` returns 1 when no supported package manager is available, letting `_ensure_node` fall through to the next install method (release tarball)
- CHANGE.md left as `[Unreleased] v0.3.7` — no tag created; k3d-manager consumer drives the release cadence via subtree pull

## Theme

A small, disciplined extraction. Two functions that were 1 over the if-count threshold got dedicated helpers extracted into them, clearing the last two entries from k3d-manager's allowlist. The changes were surgically minimal — no refactoring beyond the stated goal, no behavioral drift. Copilot flagged two style issues (indentation, missing comment) that should have been self-reviewed before push; both were fixed in one follow-up commit. The allowlist is now empty.
