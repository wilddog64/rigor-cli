# Retrospective — v0.1.4

**Date:** 2026-05-03
**Milestone:** multi-language lint via `RIGOR_LINT_BACKENDS`
**PR:** #7 — merged to main (`ac7a39d`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- The implementation stayed narrowly scoped: `_rigor_lint` replaced `_rigor_shellcheck` without changing `audit` or the lib-foundation subtree.
- The new backend mapping was easy to test with small shell stubs in BATS.
- Copilot’s review caught the only real contract mismatch: extensionless shell scripts were documented before the implementation supported them.

## What Changed

- `rigor lint` now dispatches per extension via `RIGOR_LINT_BACKENDS`.
- README and Copilot instructions now describe `lint` as rigor-cli-owned dispatch, not lib-foundation audit.
- The release branch was compacted into `v0.1.4 SHIPPED` state in the memory bank.

## Process Notes

- When `lint` stops being shell-only, docs must be updated alongside tests or Copilot will flag the mismatch.
- Release housekeeping should land on a fresh branch after merge so future work starts from a clean branch point.

## Theme

v0.1.4 expanded rigor-cli from shell-only linting into a small multi-language dispatch layer. The changes were mostly about preserving the old default while making the backend configurable, and Copilot’s review focused on the one place where the docs got ahead of the code.
