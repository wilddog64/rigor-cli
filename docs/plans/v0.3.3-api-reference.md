## Format Rules

- Use the same table structure as `k3d-manager/docs/api/functions.md`
- Descriptions: one sentence, present tense, active voice
- Signature column: show env vars and flags where relevant, omit for trivial predicates
- Do NOT document double-underscore (`__`) functions — those are internal implementation details
- Private functions with no consumer use (e.g. `_install_debian_kubernetes_client`, platform-specific sub-helpers) — group under "Internal helpers (not for direct use)" with a single-line table, no detail
- Add a "Global Variables" section at the end: `_DCRS_PROVIDER`, `_RCRS_RUNNER` — document what sets them and when they're valid
