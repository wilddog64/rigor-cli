# Spec: v0.1.1 — `mapfile` bash 3.2 compat fix

**Repo:** `wilddog64/rigor-cli`
**Branch:** `feat/v0.1.1`
**File:** `bin/rigor`

---

## Problem

`_rigor_shellcheck` uses `mapfile -t files < <(...)` to collect `.sh` files when none are
passed explicitly. `mapfile` requires bash 4+. The stated requirement is bash ≥ 3.2, so
this is a bug on any system still running the macOS system bash (3.2).

1. `git pull origin feat/v0.1.1` in `wilddog64/rigor-cli`

## Change

**Target file:** `bin/rigor`

**Old `_rigor_shellcheck`:**

```bash
_rigor_shellcheck() {
  local files=("$@")
  if [[ ${#files[@]} -eq 0 ]]; then
    mapfile -t files < <(git ls-files '*.sh' 2>/dev/null || true)
  fi
  if [[ ${#files[@]} -eq 0 ]]; then
    echo "rigor lint: no shell files provided" >&2
    return 0
  fi
  shellcheck "${files[@]}"
}
```

**New `_rigor_shellcheck`:**

```bash
_rigor_shellcheck() {
  local files=("$@")
  if [[ ${#files[@]} -eq 0 ]]; then
    while IFS= read -r f; do
      files+=("$f")
    done < <(git ls-files '*.sh' 2>/dev/null || true)
  fi
  if [[ ${#files[@]} -eq 0 ]]; then
    echo "rigor lint: no shell files provided" >&2
    return 0
  fi
  shellcheck "${files[@]}"
}
```

The `while IFS= read -r` pattern works in bash 3.2+.
