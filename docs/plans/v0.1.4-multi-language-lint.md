# Spec: v0.1.4 — Multi-Language Lint via `RIGOR_LINT_BACKENDS`

**Branch:** `feat/v0.1.4`
**Files:** `bin/rigor`, `scripts/tests/rigor.bats`, `README.md`, `.github/copilot-instructions.md`

---

## Problem

`rigor lint` is hardcoded to `shellcheck` on `.sh` files. Now that `rigor review` is
language-agnostic, rigor-cli's scope has expanded beyond shell. Consumers using Python,
TypeScript, or other languages cannot configure per-language linters — they must use
`rigor review` (AI) or fall back to language-specific tooling entirely outside rigor-cli.

**Goal:** make `rigor lint` dispatch to per-language static analysis backends configured
by an env var, while keeping the default behavior (`shellcheck` on `.sh`) identical.

---

## Design

**`RIGOR_LINT_BACKENDS`** — space-separated `ext:command` pairs.

```
RIGOR_LINT_BACKENDS="sh:shellcheck py:ruff ts:eslint"
```

- Default when unset: `sh:shellcheck`
- Each `ext:command` pair: collect files with that extension, run the command against them
- If backend binary is not installed: emit a `_warn` and skip — do not fail
- If explicit files are given to `rigor lint`, only run backends whose extension matches
- Return non-zero exit code if any backend returns non-zero

`rigor audit` is unchanged — `_agent_audit` from lib-foundation already scopes checks
by file extension (shell-only for bare sudo / tab indent / if-count; YAML-only for IP check).

```bash
_rigor_lint() {
  local explicit_files=("$@")
  local backends="${RIGOR_LINT_BACKENDS:-sh:shellcheck}"
  local exit_code=0
  local ran=0

  local pair ext cmd
  while IFS= read -r pair; do
    ext="${pair%%:*}"
    cmd="${pair##*:}"
    [[ -z "$ext" || -z "$cmd" ]] && continue

    if ! command -v "$cmd" >/dev/null 2>&1; then
      _warn "rigor lint: backend '$cmd' for .$ext not found — skipping"
      continue
    fi

    local matched=()
    if [[ ${#explicit_files[@]} -gt 0 ]]; then
      local f
      for f in "${explicit_files[@]}"; do
        [[ "$f" == *."$ext" ]] && matched+=("$f")
      done
    else
      while IFS= read -r f; do
        matched+=("$f")
      done < <(git ls-files "*.$ext" 2>/dev/null || true)
    fi

    if [[ ${#matched[@]} -gt 0 ]]; then
      ran=1
      "$cmd" "${matched[@]}" || exit_code=$?
    fi
  done < <(printf '%s\n' "$backends" | tr ' ' '\n' | grep -v '^$')

  if [[ $ran -eq 0 && ${#explicit_files[@]} -eq 0 ]]; then
    _warn "rigor lint: no files found for configured backends"
  fi

  return "$exit_code"
}

case "${1:-}" in
  checkpoint) shift; _agent_checkpoint "$@" ;;
  audit)      shift; _agent_audit "$@" ;;
  lint)       shift; _rigor_lint "$@" ;;
  review)     shift; _rigor_review "$@" ;;
  *)          _rigor_usage ;;
esac
```
