## Goal

Create a standalone CLI that exposes the three agent rigor functions as subcommands:

```
rigor checkpoint    → _agent_checkpoint
rigor audit         → _agent_audit
rigor lint [file…]  → _agent_lint [file…]
```

Any repo can install `bin/rigor` (symlink or copy) and call these commands without depending on k3d-manager.

## Step 2 — `bin/rigor`

Create `bin/rigor` (mode 0755):

```bash
#!/usr/bin/env bash
set -euo pipefail

_RIGOR_BIN_DIR="$(cd -P "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
_RIGOR_LIB_DIR="${_RIGOR_BIN_DIR}/../scripts/lib/foundation"

# shellcheck source=../scripts/lib/foundation/system.sh
source "${_RIGOR_LIB_DIR}/system.sh"
# shellcheck source=../scripts/lib/foundation/agent_rigor.sh
source "${_RIGOR_LIB_DIR}/agent_rigor.sh"

_rigor_usage() {
  echo "Usage: rigor <command> [args]"
  echo ""
  echo "Commands:"
  echo "  checkpoint          Stage all changes and create a git checkpoint commit"
  echo "  audit               Run agent audit checks on staged .sh files"
  echo "  lint [file…]        Run shellcheck on specified files (default: all .sh)"
  exit 1
}

case "${1:-}" in
  checkpoint) shift; _agent_checkpoint "$@" ;;
  audit)      shift; _agent_audit "$@" ;;
  lint)       shift; _agent_lint "$@" ;;
  *)          _rigor_usage ;;
esac
```

## Step 3 — CI (`.github/workflows/ci.yml`)

```yaml
name: CI
on:
  push:
    branches: [main, "feat/**"]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4
      - name: Install bats
        run: |
          git clone --depth 1 --branch v1.11.0 https://github.com/bats-core/bats-core.git /tmp/bats-core
          sudo /tmp/bats-core/install.sh /usr/local
      - name: shellcheck
        run: shellcheck bin/rigor
      - name: BATS
        run: |
          env -i PATH="/usr/local/bin:/usr/bin:/bin" HOME="$HOME" TMPDIR="${TMPDIR:-/tmp}" \
            bash --norc --noprofile -c 'bats scripts/tests/rigor.bats'
```

## Step 4 — BATS tests (`scripts/tests/rigor.bats`)

Tests must use a temp git repo (same pattern as lib-foundation). Exactly 3 tests required:

1. **`rigor audit` passes on clean staged file** — stage a 2-space-indented `.sh` file, run `rigor audit`, expect exit 0
2. **`rigor audit` fails on tab-indented staged file** — stage a tab-indented `.sh` file, expect exit non-zero
3. **`rigor lint` exits non-zero on shellcheck violation** — create a `.sh` with `$var` unquoted, run `rigor lint <file>`, expect exit non-zero

For setup, each test must:
- Create a temp dir with `mktemp -d`
- `git init` inside it
- `git config user.email` + `git config user.name`
- Copy `bin/rigor` and `scripts/lib/foundation/` into the temp repo so the path resolution works
- Run `rigor` from within the temp repo

Teardown: `rm -rf "$BATS_TEST_TMPDIR"`.

## Step 5 — `README.md`

````markdown
# rigor-cli

Standalone CLI for the [lib-foundation](https://github.com/wilddog64/lib-foundation) agent rigor framework.

## Install

```bash
# Add to your repo as a git subtree (recommended — includes scripts/lib/foundation/)
git subtree add --prefix=.rigor \
  https://github.com/wilddog64/rigor-cli.git main --squash

# Note: copying only bin/rigor is not sufficient — bin/rigor sources
# ../scripts/lib/foundation/system.sh relative to its location.
# Use the subtree method above, or copy both bin/rigor and scripts/lib/foundation/.
```

## Usage

```bash
rigor checkpoint          # stage all + commit checkpoint
rigor audit               # check staged .sh files (runs pre-commit checks)
rigor lint [file…]        # shellcheck on specified files
```

## Requirements

- bash ≥ 3.2
- git
- shellcheck (for `rigor lint`)

## Consumers

- [k3d-manager](https://github.com/wilddog64/k3d-manager)
````
