## Problem

`_agent_lint` (line 220) only collects staged `*.sh` files:

```bash
staged_files="$(git diff --cached --name-only --diff-filter=ACM -- '*.sh' 2>/dev/null || true)"
```

Projects that use lib-foundation now also commit `.js` files (Playwright automation)
and `.md` docs. The lint gate silently skips those, so architectural violations in
JS or rule violations in docs never reach the AI reviewer.

Additionally, the prompt on line 232 says "staged **shell** files" — inaccurate once
the glob covers multiple types.

---

## Fix

### Change 1 — Expand staged-files glob

**Exact old line (line 220):**

```bash
   staged_files="$(git diff --cached --name-only --diff-filter=ACM -- '*.sh' 2>/dev/null || true)"
```

**Exact new line:**

```bash
   staged_files="$(git diff --cached --name-only --diff-filter=ACM -- '*.sh' '*.js' '*.md' 2>/dev/null || true)"
```

### Change 2 — Fix prompt wording

**Exact old line (line 232):**

```bash
   prompt="Review the following staged shell files for architectural violations.\n\nRules:\n$(cat "$rules_file")\n\nFiles:\n$staged_files"
```

**Exact new line:**

```bash
   prompt="Review the following staged files for architectural violations.\n\nRules:\n$(cat "$rules_file")\n\nFiles:\n$staged_files"
```
