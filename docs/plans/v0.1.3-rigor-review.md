## Problem

`_ai_agent_review` (lib-foundation `system.sh`) is already a generic AI dispatch wrapper,
but rigor-cli exposes no CLI entrypoint for it. Any project wanting AI review must either
source lib-foundation manually or copy k3d-manager's `copilot_draft_spec` pattern. A
`rigor review` subcommand makes it available to any project that has rigor-cli installed,
with no extra wiring.

  echo "  review [--prompt <text>] [args‚Ä¶]    Run AI review via configured backend (default: Copilot)"

_rigor_review() {
  local repo_root
  repo_root="$(git rev-parse --show-toplevel 2>/dev/null || true)"
  if [[ -n "$repo_root" && ! -f "$repo_root/.github/copilot-instructions.md" ]]; then
    _warn "[rigor review] .github/copilot-instructions.md not found ‚Äî Copilot will use default instructions"
  fi
  _ai_agent_review "$@"
}

## How it works

`_rigor_review` checks for `.github/copilot-instructions.md` at the calling project's git
root and warns if missing (Copilot still runs, just with default instructions). It then
delegates to `_ai_agent_review`, which dispatches to the backend selected by `AI_REVIEW_FUNC`
(default: `copilot`) with the model from `AI_REVIEW_MODEL` (default: `gpt-5.4-mini`).

Because `_copilot_review` already `cd`s to the git root before invoking Copilot CLI, the
calling project's `copilot-instructions.md` is picked up automatically ‚Äî no extra wiring.

**Usage examples:**
```bash
rigor review --prompt "review this diff for security issues"
rigor review --prompt "check this function for shell injection" --model claude-sonnet-4-6
AI_REVIEW_FUNC=copilot rigor review --prompt "draft a spec for this bug"
```
