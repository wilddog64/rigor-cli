## Problem

`_agent_audit` currently checks only `.sh` and `.bats` files. Staged `.yaml`/`.yml`
files (Helm values, Kubernetes manifests, config maps) may contain hardcoded IPv4
addresses instead of CoreDNS hostnames. These slip through commit-time checks
and surface as environment-specific failures or security policy violations.

---

## Goal

Add a staged `.yaml`/`.yml` file check to `_agent_audit` that:
1. Detects hardcoded IPv4 addresses in staged `.yaml` / `.yml` file content
2. Warns with a suggestion to use a CoreDNS hostname instead
3. Sets `status=1` on any violation (blocking the commit)

## Implementation

### New block in `_agent_audit` — insert after line 173 (after the kubectl exec check, before `return "$status"`)

**Exact old code** (the final two lines of `_agent_audit` before the return):

```bash
   fi

   return "$status"
}
```

**Exact new code** — replace those two lines with:

```bash
   fi

   local changed_yaml
   changed_yaml="$(
      git diff --cached --name-only -- '*.yaml' '*.yml' 2>/dev/null || true
   )"
   if [[ -n "$changed_yaml" ]]; then
      local file
      for file in $changed_yaml; do
         [[ -f "$file" ]] || continue
         local ip_lines
         ip_lines=$(git show :"$file" 2>/dev/null \
            | grep -En '[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}' || true)
         if [[ -n "$ip_lines" ]]; then
            _warn "Agent audit: hardcoded IP address in $file — use a CoreDNS hostname instead:"
            _warn "$ip_lines"
            status=1
         fi
      done
   fi

   return "$status"
}
```

### Explanation

- `git diff --cached --name-only -- '*.yaml' '*.yml'` — staged YAML files only
- `git show :"$file"` — reads the staged (index) version, not the working tree
- `grep -En '[0-9]{1,3}...'` — matches any IPv4-looking string with line numbers
- Pattern is intentionally broad (catches `192.168.x.x`, `10.0.x.x`, `172.x.x.x`, and any other dotted-quad); false positives (e.g. chart version `1.2.3.4`) are acceptable trade-offs — the developer can review and override
- Warning suggests CoreDNS hostname as the preferred alternative

## BATS Test

### Add to `scripts/tests/lib/agent_rigor.bats`

Append these two tests after the existing `_agent_audit` tests:

```bash
@test "_agent_audit: passes when staged yaml has no hardcoded IP" {
  local repo
  repo="$(mktemp -d)"
  git -C "$repo" init -q
  git -C "$repo" config user.email "test@example.com"
  git -C "$repo" config user.name "Test"
  printf 'host: my-service.default.svc.cluster.local\n' > "$repo/values.yaml"
  git -C "$repo" add values.yaml
  (
    cd "$repo"
    run _agent_audit
    [ "$status" -eq 0 ]
  )
  rm -rf "$repo"
}

@test "_agent_audit: fails when staged yaml contains hardcoded IP" {
  local repo
  repo="$(mktemp -d)"
  git -C "$repo" init -q
  git -C "$repo" config user.email "test@example.com"
  git -C "$repo" config user.name "Test"
  printf 'host: 192.168.1.100\n' > "$repo/values.yaml"
  git -C "$repo" add values.yaml
  (
    cd "$repo"
    run _agent_audit
    [ "$status" -eq 1 ]
    [[ "$output" == *"hardcoded IP"* ]]
  )
  rm -rf "$repo"
}
```
