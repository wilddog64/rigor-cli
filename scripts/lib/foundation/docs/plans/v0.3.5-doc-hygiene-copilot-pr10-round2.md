## Context

Three Copilot findings from the second review of PR #10 (all on `d00bccb`):

- **Finding 1** (`doc_hygiene.sh:27`) — `_DHC_STAGED` is global; should be `local`
- **Finding 2** (`doc_hygiene.sh:40`) — `[[ -f "$file" ]]` can give false negatives in staged mode (file deleted from working tree after staging); use `git cat-file -e` instead
- **Finding 3** (`doc_hygiene.bats:116`) — test "violation in staged content is caught even when working tree is clean" only tests the explicit-path (working-tree) path; never exercises the no-args/staged mode

---

## The Fix

### File 1: `scripts/lib/doc_hygiene.sh`

**Fix 1 — make `_DHC_STAGED` local (line 27)**

Old:
```bash
  local status=0
  _DHC_STAGED=0
```

New:
```bash
  local status=0
  local _DHC_STAGED=0
```

**Fix 2 — use `git cat-file -e` in staged mode (line 40)**

Old:
```bash
      [[ -f "$file" ]] || continue
```

New:
```bash
      if [[ "${_DHC_STAGED:-0}" -eq 1 ]]; then
         git cat-file -e :"$file" 2>/dev/null || continue
      else
         [[ -f "$file" ]] || continue
      fi
```

---

### File 2: `scripts/tests/lib/doc_hygiene.bats`

**Fix 3 — replace the staged-content test to actually exercise the no-args/staged mode**

Old (lines 116–128):
```bash
@test "violation in staged content is caught even when working tree is clean" {
   # Stage a file with a placeholder URL
   echo "See [repo](https://github.com/user/myrepo)" > "$TEST_DIR/staged.md"
   git -C "$TEST_DIR" init -q
   git -C "$TEST_DIR" config user.email "test@test.com"
   git -C "$TEST_DIR" config user.name "Test"
   git -C "$TEST_DIR" add staged.md
   # Fix the working-tree copy AFTER staging — staged content still has violation
   echo "See [repo](https://github.com/wilddog64/myrepo)" > "$TEST_DIR/staged.md"
   # Call with explicit path (working-tree mode) — should PASS (working tree is clean)
   run _doc_hygiene_check "$TEST_DIR/staged.md"
   [ "$status" -eq 0 ]
}
```

New (replace the entire test block — same name, different body):
```bash
@test "violation in staged content is caught even when working tree is clean" {
   git -C "$TEST_DIR" init -q
   git -C "$TEST_DIR" config user.email "test@test.com"
   git -C "$TEST_DIR" config user.name "Test"
   # Stage a file with a placeholder URL
   echo "See [repo](https://github.com/user/myrepo)" > "$TEST_DIR/staged.md"
   git -C "$TEST_DIR" add staged.md
   # Fix the working-tree copy AFTER staging — staged content still has violation
   echo "See [repo](https://github.com/wilddog64/myrepo)" > "$TEST_DIR/staged.md"
   # Run with no args from inside the repo — staged mode, should FAIL (index has violation)
   local prev_dir="$PWD"
   cd "$TEST_DIR"
   run _doc_hygiene_check
   cd "$prev_dir"
   [ "$status" -eq 1 ]
   [[ "$output" == *"placeholder URL"* ]]
}
```
