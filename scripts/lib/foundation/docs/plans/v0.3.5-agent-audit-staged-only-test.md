## Context

PR #11 reverted `_agent_audit`'s `changed_sh` collection from staged+unstaged to staged-only:

```bash
# After PR #11 (current main):
changed_sh="$(
   git diff --cached --name-only -- '*.sh' 2>/dev/null || true
)"
```

No BATS test verifies this behavioral contract — a file with bare `sudo` that is **not staged** must NOT trigger the audit. This spec adds that test.

---

## The Fix — add one test to `agent_rigor.bats`

**File:** `scripts/tests/lib/agent_rigor.bats`

Append the following test **after** the last `@test` block (after line 190, the closing `}` of `_agent_audit fails when if-count exceeds threshold`):

### Old (end of file — line 190 onward)

```
  [ "$status" -ne 0 ]
  [[ "$output" == *"exceeds if-count threshold"* ]]
}
```

### New (same closing lines + new test appended)

```
  [ "$status" -ne 0 ]
  [[ "$output" == *"exceeds if-count threshold"* ]]
}

@test "_agent_audit ignores unstaged .sh changes" {
  mkdir -p scripts
  cat <<'SCRIPT' > scripts/unstaged.sh
function clean() {
   echo ok
}
SCRIPT
  git add scripts/unstaged.sh
  git commit -m "add unstaged" >/dev/null
  # Add bare sudo to the file but do NOT stage it
  cat <<'SCRIPT' >> scripts/unstaged.sh
function needs_sudo() {
   sudo rm -rf /tmp/test
}
SCRIPT
  # File has bare sudo but is NOT staged — audit must pass
  run _agent_audit
  [ "$status" -eq 0 ]
}
```
