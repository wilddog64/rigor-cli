## Context

Copilot PR #12 finding: `_dh_strip_fences` only matches fences starting at column 1
(`/^[`~]{3}/`). Markdown fenced blocks inside list items are indented (e.g. 3 spaces +
triple-backtick). The current awk pattern misses them — `http://` inside an indented
fence still triggers Check 2 (false positive).

---

## The Fix

### File 1: `scripts/lib/doc_hygiene.sh`

**Old** (lines 13–18 — `_dh_strip_fences`):

```bash
_dh_strip_fences() {
  awk 'BEGIN{f=0}
       /^[`~]{3}/{f=!f; print ""; next}
       f{print ""}
       !f{print}'
}
```

**New** (match optional leading whitespace before fence marker; track opening fence char
to correctly close mismatched fence types):

```bash
_dh_strip_fences() {
   awk 'BEGIN{in_fence=0; fence_char=""}
        {
          if ($0 ~ /^[[:space:]]*[`~]{3}/) {
            i = 1
            while (i <= length($0) && substr($0, i, 1) ~ /[ \t]/) i++
            c = substr($0, i, 1)
            if (!in_fence) {
              in_fence = 1; fence_char = c; print ""; next
            } else if (c == fence_char) {
              in_fence = 0; fence_char = ""; print ""; next
            }
          }
          if (in_fence) { print "" } else { print }
        }'
}
```
