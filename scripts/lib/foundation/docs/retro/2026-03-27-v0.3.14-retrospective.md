# Retrospective — v0.3.14

**Date:** 2026-03-27
**Milestone:** Copilot PR #51 deferred fixes — agy binary, curl fast-fail, NUL loops, doc fixes
**PR:** #20 — squash-merged to main (`bbbaf053`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- All 5 deferred findings from k3d-manager Copilot PR #51 landed cleanly: `_ensure_antigravity_ide` agy detection, `_antigravity_browser_ready` curl fast-fail, all three `_agent_audit` loops NUL-safe, `docs/api/functions.md` accuracy fix, `CHANGE.md` version entries
- Codex implemented all 5 fixes in a single commit (`e52b819`) with shellcheck clean and 78 BATS passing
- Copilot caught real issues across 3 review rounds — NUL loop gaps, weak BATS test, spec contradiction, `_err` stub that didn't actually exit, retro wording inaccuracy
- Each Copilot round was addressed and resolved in one fix commit — no back-and-forth within a round

## What Went Wrong

- Spec initially said "Do NOT modify files outside 4 targets + 2 BATS files" but Definition of Done required memory-bank updates — contradiction caught by Copilot (round 3), not noticed before handoff
- BATS test for `_antigravity_browser_ready` curl fast-fail used `_err() { return 1; }` — because stub returned instead of exiting, the test passed even though the function fell through to the sleep loop; caught by Copilot (round 4)
- NUL loop fix in `e52b819` only converted the tab-scan loop; if-count and bare-sudo loops were missed; caught by Copilot (round 1), required second fix commit (`7a662e1`)
- Retro wording described `_curl` as "silently calls `exit 1`" — inaccurate because `_err` prints before exiting; caught by Copilot (round 4)

## Process Rules Added

| Rule | Where |
|---|---|
| `_err` stubs in BATS must use `exit 1`, not `return 1` — `_err` exits in production | Add to spec template: BATS Rules section |
| Spec "What NOT To Do" must explicitly list all permitted files including memory-bank | `/handoff` skill spec review gate |
| When converting loops to NUL-delimited, audit ALL loops in the function — not just the one mentioned | Agent instruction note |

## Decisions Made

- All three `_agent_audit` file loops now use `while IFS= read -r -d '' file; done < <(git diff --cached --name-only -z -- '*.sh')` — consistent across if-count, bare-sudo, and tab-scan checks
- `_ensure_antigravity_ide` checks `agy || antigravity` at every detection point — Homebrew installs `agy`, not `antigravity`
- `_antigravity_browser_ready` fast-fails with `_err` when `curl` is missing — guard added before the poll loop

## Theme

v0.3.14 was a deferred-debt release: 5 findings caught by Copilot during k3d-manager PR #51 that lived in subtree-managed files, handed off to Codex, then put through 4 Copilot review rounds. The implementation was correct from the start; the fixes in rounds 2-4 were test quality and documentation accuracy. The pattern held: Copilot catches the subtle things (stub semantics, spec contradictions, wording precision) that implementation review misses.
