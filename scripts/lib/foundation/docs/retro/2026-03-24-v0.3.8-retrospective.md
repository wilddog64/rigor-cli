# Retrospective — v0.3.8

**Date:** 2026-03-24
**Milestone:** _agent_audit tab indentation enforcement
**PR:** #14 — merged to main (`a669a63`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Codex implemented the spec exactly — correct insertion point, correct grep pattern, correct BATS tests on first attempt
- shellcheck clean on first commit; no follow-up fixes needed for core implementation
- Copilot caught three real issues: spec version inconsistency, memory-bank factual error (v0.3.7 tag status), and incomplete grep pattern (`^\t` → `^ *\t` for mixed indentation)
- The mixed space+tab finding from Copilot was a genuine improvement — pure `^\t` would have missed a common real-world case
- Retro-driven correction: memory-bank said v0.3.7 was tagged when it wasn't; Copilot caught the inconsistency by comparing retro vs memory-bank

## What Went Wrong

- Spec filename used v0.3.9 but branch/content used v0.3.8 — version numbering was ambiguous from the start; should have decided the version before naming the file
- Memory-bank was incorrectly updated to say v0.3.7 was tagged/released — copied from a pattern without verifying against actual git tags

## Process Rules Added

| Rule | Where |
|---|---|
| Verify git tags with `git tag -l` before writing memory-bank entries about release status | post-merge checklist |
| Spec filename must match the version it ships in — decide version before creating the file | spec authoring checklist |

## Decisions Made

- Tab check uses `$'^ *\t'` (optional spaces + tab) not `$'^\t'` (tab-only) — catches mixed indentation which is the more common real-world violation
- Makefile exclusion is implicit — `_agent_audit` only checks `*.sh` via `changed_sh`; no explicit exclusion needed
- Gists for agent rigor framework deferred to v0.9.16+ k3d-manager milestone — stable enough today but no pressing audience until article is written

## Theme

A small code quality enforcement PR that validated the full spec → Codex → Copilot review pipeline working correctly. The core implementation was clean on first try. Copilot's value was in catching cross-document inconsistencies (spec version, memory-bank accuracy) rather than functional bugs — exactly the kind of review humans tend to skip. The mixed indentation grep fix was the only real functional improvement.
