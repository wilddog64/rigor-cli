# Retrospective — v0.3.13

**Date:** 2026-03-25
**Milestone:** Fix `_antigravity_browser_ready` curl probe
**PR:** #19 — squash-merged to main (`e870c6d9`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Single-line fix was correctly isolated: spec, implementation, and test update all scoped to one function
- Copilot caught the missing `--max-time` timeout gap after removing `_curl` wrapper — the fix preserved `CURL_MAX_TIME` semantics
- BATS tests updated in the same commit as the implementation fix (no stale stubs)
- CI went green after one fix commit addressing Copilot's finding

## What Went Wrong

- Root cause originated in v0.3.12: `_curl` was shipped as a boolean probe in `_antigravity_browser_ready` without recognizing that `_curl` calls `_run_command --quiet` without `--soft`, meaning any non-zero exit terminates the script
- Gemini discovered the bug in k3d-manager before it was caught in lib-foundation — the lib-foundation BATS tests stubbed `_curl` entirely, masking the probe behavior
- `_curl` wrapper pattern is a silent footgun for boolean probing — not documented in the wrapper, nothing in code style prevents misuse

## Process Rules Added

- **`_curl` must NOT be used as a boolean probe** — add to AGENTS.md/CLAUDE.md: use `_run_command --soft -- curl ...` for port checks; `_curl` is output-capturing only

## Decisions Made

- `_antigravity_browser_ready` poll uses `_run_command --soft -- curl --max-time "${CURL_MAX_TIME:-30}" -sf` — explicitly preserves the `CURL_MAX_TIME` env var that `_curl` wrapper would have applied implicitly
- BATS stubs now target `_run_command` (not `_curl`) for probe tests — reflects the real call path

## Theme

v0.3.13 was a single-line hotfix surfaced by live testing in k3d-manager. The underlying issue is a subtle API footgun in `_curl` — on non-zero exit it terminates via `_err` (invoked from `_run_command --quiet`), which makes it unsafe for boolean probing because the script exits instead of returning a status. The fix is correct and the BATS coverage is now honest. The lesson: wrapper functions that call `exit` must be documented as such; callers must never use them in `if` conditions.
