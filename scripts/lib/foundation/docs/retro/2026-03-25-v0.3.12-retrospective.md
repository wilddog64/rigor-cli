# Retrospective — v0.3.12

**Date:** 2026-03-25
**Milestone:** Antigravity IDE install + Playwright MCP config helpers
**PR:** #18 — squash-merged to main (`91340d62`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- Spec-to-implementation flow worked cleanly: spec written in one session, Codex implemented in the next
- Copilot caught 5 real issues: missing `apt-get update`, `mktemp` portability, floating `@latest` version, bare `curl`, no BATS coverage
- All 5 Copilot findings addressed in a single fix commit (`9f28d88`) before merge
- `PLAYWRIGHT_MCP_VERSION` env var gives callers version override without changing function signature — good extensibility pattern

## What Went Wrong

- Codex fabricated SHA `ae0e8b9` as the implementation commit; actual commit SHA needed manual verification via `git log origin/feat/v0.3.12`
- Post-merge steps (tag, release, branch cut) were executed without pausing for user review — should have confirmed before proceeding
- BATS test run blocked by `_smartcd` shell hook pollution; couldn't verify locally before commit (relied on CI)

## Process Rules Added

- **Pause before merge/release** — confirm with user before running `/post-merge` steps; do not chain merge → tag → release in one pass without a checkpoint

## Decisions Made

- `_ensure_antigravity_ide` uses best-effort apt/dnf with clean failure on unsupported platforms — same pattern as `_ensure_node`; no hard failure for unrecognised distros
- Playwright MCP version defaulted via `PLAYWRIGHT_MCP_VERSION` env var (default `0.0.26`) rather than hardcoded string — allows consumers to pin without changing lib code
- `_antigravity_browser_ready` polls port 9222 with configurable timeout; shipped in v0.3.12 using `_curl` wrapper — refined in v0.3.13 to use `_run_command --soft -- curl` after `_curl` was identified as the root cause of silent `exit 1` on probe failure

## Theme

v0.3.12 was a short, focused sprint: three new helpers to support the Antigravity × Playwright MCP workflow that will eventually enable browser automation in the agent loop. The implementation was straightforward; the value came from Copilot's review catching the portability and correctness gaps that Codex left behind. Post-merge housekeeping moved too fast this session — the process rule added reflects that.
