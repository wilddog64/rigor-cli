# Retrospective — v0.3.11

**Date:** 2026-03-25
**Milestone:** v0.3.11 — `_agent_audit` YAML hardcoded-IP check
**PR:** #17 — merged to main (`2625683`)
**Participants:** Claude, Codex, Copilot

## What Went Well

- **Codex implementation clean** — spec had exact old/new code blocks; Codex produced a correct first commit with zero shellcheck warnings and 17/17 BATS passing
- **Copilot review caught 7 real issues** — NUL-delimited loop, `--diff-filter=ACM`, trap cleanup, `.yml` test gap, spec path errors, doc typo; all substantive
- **Same-session resolution** — all 7 Copilot findings fixed, replied, and resolved without needing a second review cycle
- **CI fast** — both CI runs (initial + fix commit) completed in 12–13 seconds

## What Went Wrong

- **`for file in $changed_yaml` without `-z`** — initial implementation used word-splitting on newline-delimited output; should default to NUL-delimited for all new file-iteration loops
- **No `.yml`-specific test** — spec only included `.yaml` tests; Copilot caught the gap; `.yml` should have been in the original spec
- **Wrong BATS path in spec** — spec said `scripts/tests/agent_rigor.bats`; actual path is `scripts/tests/lib/agent_rigor.bats`; Copilot caught two occurrences
- **No trap for temp dir cleanup** — both YAML tests used `rm -rf "$repo"` at the end without a `RETURN` trap; leaks on assertion failure

## Process Rules Added

| Rule | Where |
|---|---|
| New file-iteration loops in `_agent_audit` must use `-z` + `while IFS= read -r -d ''` | AGENTS.md, spec template |
| BATS tests creating temp dirs must use `trap 'rm -rf "$dir"' RETURN` immediately after `mktemp -d` | BATS test template |
| Specs covering multiple file extensions must include at least one test per extension | spec template |
| Spec BATS section must reference `scripts/tests/lib/` not `scripts/tests/` | spec template |

## Decisions Made

- **Pattern is intentionally broad** — any dotted-quad IPv4 match fails; false positives (chart version strings like `1.2.3.4`) are acceptable; developer reviews the warning
- **`--diff-filter=ACM`** — only Added/Copied/Modified files scanned; deletes and renames excluded; this replaces the `-f` worktree guard
- **Scope: `_agent_audit` only** — YAML IP check belongs in pre-commit enforcement, not `_agent_lint` (AI-assisted) or CI only

## Theme

v0.3.11 was a focused one-function addition: staged YAML files now get checked for hardcoded IPv4 addresses alongside the existing shell checks. Codex implemented it cleanly from spec on the first try. Copilot's 7-finding review was the most substantive of any lib-foundation PR to date — every finding was valid and improved the code. The biggest process gaps were in the spec template (wrong BATS path, missing `.yml` test, no trap reminder) rather than in the implementation itself. All gaps are now documented as process rules for the next spec author.
